ClusterResourceQuota

This OpenShift-specific object manages resource usage across **multiple namespaces**. It sets an aggregate quota for a group of namespaces (e.g., a whole department), preventing one team from consuming an unfair share of cluster resources.

Node Level Control

At this level, control is about the total capacity of the node itself and its `allocatable` resources. Platform administrators can also use **Taints and Tolerations** to dedicate nodes to specific types of workloads (e.g., GPU-enabled nodes), ensuring that only pods with the correct `toleration` can be scheduled there. This acts as a powerful isolation mechanism.

ResourceQuota

A standard Kubernetes object providing constraints on a **single namespace**. It limits the total amount of resources (CPU, memory) and number of objects (pods, services) that can exist within that namespace.

LimitRange

This object operates within a namespace and defines constraints on **individual pods or containers**. It's used to set default request/limit values, and enforce minimum/maximum resource values per container.

Container Requests & Limits

The most fundamental level, defined directly in a pod spec. A **request** is guaranteed by Kubernetes for scheduling. A **limit** is the maximum a container can use. Exceeding a memory limit results in termination (OOMKilled).

Resource Management as Code: OpenShift Templates

As shown in the OpenShift workshop documentation, `Templates` are a powerful feature for bundling application components with their required resource control policies into a single, reusable blueprint.

Example Template with Embedded Quota

This example shows how a `Template` can define a `ResourceQuota` object alongside the application's `DeploymentConfig`. When this template is processed, both the application and its specific resource constraints are created together, ensuring controls are applied automatically.

apiVersion: v1
kind: Template
metadata:
  name: application-with-quota-template
objects:
- apiVersion: v1
  kind: ResourceQuota
  metadata:
    name: my-app-quota
  spec:
    hard:
      requests.cpu: "1"
      limits.cpu: "2"
      requests.memory: 1Gi
      limits.memory: 2Gi
- apiVersion: apps.openshift.io/v1
  kind: DeploymentConfig
  metadata:
    name: my-application
  spec:
    # ... application deployment details go here
    template:
      spec:
        containers:
        - name: my-app-container
          image: my-image
          resources:
            requests:
              cpu: 500m
              memory: 512Mi
            limits:
              cpu: "1"
              memory: 1Gi
parameters:
# ... template parameters can be defined here

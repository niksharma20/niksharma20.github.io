Blueprint: Horizontal Scaling (Single Cluster)

The Horizontal Pod Autoscaler (HPA) is a core Kubernetes component that embodies cloud-native elasticity. It automatically adjusts the number of pod replicas based on observed metrics, ensuring performance under load while controlling costs.

The HPA Control Loop

The HPA continuously compares observed metrics (like CPU utilization) against a target you define. Use the slider below to see how a change in the current metric affects the calculated number of desired replicas, based on the core HPA formula.

Current Replicas: 3 Desired Replicas: 3

Formula: `desired = ceil(current * (currentMetric / desiredMetric))`

Scaling Metrics

While CPU and memory are common, HPA can scale on more advanced metrics for greater accuracy. This requires a metrics adapter like the Prometheus adapter.

Custom Metrics

Metrics associated directly with pods, like `http_requests_per_second`. Ideal for I/O-bound workloads.

External Metrics

Metrics from outside the cluster, like the depth of an AWS SQS queue. Perfect for scaling consumer workloads.
